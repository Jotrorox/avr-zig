The DS1302 driver communicates with a DS1302 real-time clock chip using a 3-wire serial protocol (RST, IO, CLK). It returns a comptime-generic struct type parameterized by the three control pins.

```zig
const avr = @import("avr_zig");
const ds1302 = avr.drivers.sensor.ds1302;
```

## Types

### `Error`

```zig
pub const Error = error{
    InvalidDateTime,
};
```

| Variant | Meaning |
|---|---|
| `InvalidDateTime` | A field in the `DateTime` is out of range |

### `DateTime`

```zig
pub const DateTime = struct {
    second: u8,
    minute: u8,
    hour: u8,
    day: u8,
    month: u8,
    weekday: u8,
    year: u16,
};
```

| Field | Type | Valid range |
|---|---|---|
| `second` | `u8` | 0--59 |
| `minute` | `u8` | 0--59 |
| `hour` | `u8` | 0--23 (24-hour format) |
| `day` | `u8` | 1--28/29/30/31 (depends on month and year) |
| `month` | `u8` | 1--12 |
| `weekday` | `u8` | 1--7 |
| `year` | `u16` | 2000--2099 |

:::note
The DS1302 stores a 2-digit year internally. The driver adds a base of 2000, so the supported range is 2000--2099.
:::

## `Device`

```zig
pub fn Device(
    comptime rst_pin: gpio.Pin,
    comptime io_pin: gpio.Pin,
    comptime clk_pin: gpio.Pin,
) type
```

Returns a DS1302 driver type for the given pin assignment. The three pins are configured as outputs during initialization.

| Parameter | Type | Description |
|---|---|---|
| `rst_pin` | `gpio.Pin` | Chip-enable / reset pin (comptime) |
| `io_pin` | `gpio.Pin` | Bidirectional data pin (comptime) |
| `clk_pin` | `gpio.Pin` | Serial clock pin (comptime) |

**Compile constraints:**

- All three pins must be distinct. Passing the same pin for any two parameters produces a compile error: `"DS1302 RST, IO, and CLK pins must be different"`.

### Methods

#### `init`

```zig
pub fn init(self: *Self) void
```

Configures the RST, IO, and CLK pins as outputs and drives them low.

#### `readDateTime`

```zig
pub fn readDateTime(self: *Self) DateTime
```

Reads the current date and time from the RTC using a clock-burst read. Calls `init` internally to ensure pin state. The hour value is decoded from both 12-hour and 24-hour BCD formats to a 0--23 value.

**Returns:** a `DateTime` struct with the current date and time.

#### `writeDateTime`

```zig
pub fn writeDateTime(self: *Self, date_time: DateTime) Error!void
```

Writes a date and time to the RTC. Validates all fields before writing and temporarily disables write protection for the transaction.

| Parameter | Type | Description |
|---|---|---|
| `date_time` | `DateTime` | The date and time to set |

**Returns:** `Error!void` -- returns `InvalidDateTime` if any field is out of range.

**Validation rules:**

- `second` must be 0--59
- `minute` must be 0--59
- `hour` must be 0--23
- `day` must be 1 through the number of days in the given month/year (leap years handled)
- `month` must be 1--12
- `weekday` must be 1--7
- `year` must be 2000--2099

#### `setWriteProtect`

```zig
pub fn setWriteProtect(self: *Self, enabled: bool) Error!void
```

Enables or disables write protection on the RTC's register file.

| Parameter | Type | Description |
|---|---|---|
| `enabled` | `bool` | `true` to enable write protection, `false` to disable |

## Example

```zig
const avr = @import("avr_zig");
const uart = avr.uart;
const ds1302 = avr.drivers.sensor.ds1302;

const Rtc = ds1302.Device(.D2, .D3, .D4);

pub fn main() void {
    uart.init(115200);

    var rtc = Rtc{};
    rtc.init();

    // Set the time once
    rtc.writeDateTime(.{
        .year = 2026,
        .month = 3,
        .day = 16,
        .weekday = 1,
        .hour = 12,
        .minute = 0,
        .second = 0,
    }) catch {};

    // Read and print
    const dt = rtc.readDateTime();
    uart.write(dt.year);
    uart.write("-");
    uart.write(dt.month);
    uart.write("-");
    uart.write(dt.day);
    uart.write("\r\n");
}
```
