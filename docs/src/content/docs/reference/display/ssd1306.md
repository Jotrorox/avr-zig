The SSD1306 driver provides a framebuffer-backed API for monochrome OLED displays connected over I2C. It supports pixel, line, rectangle, and text drawing.

```zig
const avr = @import("avr_zig");
const ssd1306 = avr.drivers.display.ssd1306;
```

:::note
You must call `avr.i2c.init()` before using this driver.
:::

## Constants

### `default_address`

```zig
pub const default_address: u7 = 0x3C;
```

The most common I2C address for SSD1306 modules. Some boards use `0x3D` instead -- check your module's documentation or run an [I2C scan](/reference/hal/i2c/).

### `default_font`

```zig
pub const default_font = fonts.mono5x7;
```

A built-in monospaced 5x7 pixel font covering uppercase ASCII characters (space through underscore, codes 32--95). Lowercase letters are automatically converted to uppercase.

## Types

### `Color`

```zig
pub const Color = enum(u1) { off = 0, on = 1 };
```

| Variant | Meaning |
|---|---|
| `.off` | Pixel off (black) |
| `.on` | Pixel on (white) |

### `Font`

```zig
pub const Font = struct {
    glyph_width: u8,
    glyph_height: u8,
    first_char: u8,
    glyph_count: u8,
    spacing_x: u8 = 1,
    spacing_y: u8 = 1,
    fallback_char: u8 = '?',
    data: []const u8,
};
```

Describes a bitmap font. The built-in `fonts.mono5x7` is provided, but you can define your own by creating a `Font` with matching bitmap data.

| Field | Type | Description |
|---|---|---|
| `glyph_width` | `u8` | Width of each glyph in pixels |
| `glyph_height` | `u8` | Height of each glyph in pixels |
| `first_char` | `u8` | ASCII code of the first character in the font |
| `glyph_count` | `u8` | Number of glyphs in the font |
| `spacing_x` | `u8` | Horizontal spacing between characters (default 1) |
| `spacing_y` | `u8` | Vertical spacing between lines (default 1) |
| `fallback_char` | `u8` | Character shown for unsupported codes (default `'?'`) |
| `data` | `[]const u8` | Raw glyph bitmap data |

#### Font methods

| Method | Signature | Description |
|---|---|---|
| `bytesPerColumn` | `fn (Font) usize` | Bytes used per column in a glyph |
| `glyphStride` | `fn (Font) usize` | Total bytes per glyph |
| `lineAdvance` | `fn (Font) u8` | Vertical distance between text lines |
| `measureText` | `fn (Font, []const u8) u16` | Width in pixels of a text string |
| `glyph` | `fn (Font, u8) [*]const u8` | Pointer to a character's bitmap data |

### `fonts`

```zig
pub const fonts = struct {
    pub const mono5x7: Font;
};
```

Built-in font collection. Currently contains one font:

- **`mono5x7`** -- 5 pixels wide, 7 pixels tall, covers ASCII 32--95 (space, digits, uppercase letters, punctuation). Lowercase input is automatically uppercased.

## `Display`

```zig
pub fn Display(comptime display_width: u8, comptime display_height: u8) type
```

Returns a display driver type parameterized by the screen dimensions. Common sizes are 128x64 and 128x32.

| Parameter | Type | Description |
|---|---|---|
| `display_width` | `u8` | Screen width in pixels (comptime) |
| `display_height` | `u8` | Screen height in pixels (comptime) |

**Compile constraints:**

- Both dimensions must be greater than zero
- `display_height` must be divisible by 8

### Type constants

| Constant | Type | Description |
|---|---|---|
| `width` | `u8` | The display width in pixels |
| `height` | `u8` | The display height in pixels |
| `pages` | `usize` | Number of 8-pixel pages (`height / 8`) |
| `framebuffer_len` | `usize` | Size of the framebuffer in bytes (`width * pages`) |

### Fields

| Field | Type | Default | Description |
|---|---|---|---|
| `address` | `u7` | `0x3C` | I2C address of the display |
| `buffer` | `[framebuffer_len]u8` | all zeros | In-memory framebuffer |

### Methods

#### `init`

```zig
pub fn init(self: *Self) bool
```

Sends the initialization command sequence to the display and flushes the framebuffer. Returns `true` on success.

#### `clear`

```zig
pub fn clear(self: *Self, color: Color) void
```

Fills the entire framebuffer with the given color. Call `present()` afterward to update the screen.

#### `drawPixel`

```zig
pub fn drawPixel(self: *Self, x: u8, y: u8, color: Color) void
```

Sets a single pixel in the framebuffer. Out-of-bounds coordinates are silently ignored.

#### `drawLine`

```zig
pub fn drawLine(self: *Self, x0: i16, y0: i16, x1: i16, y1: i16, color: Color) void
```

Draws a line between two points using Bresenham's algorithm. Coordinates may be negative (off-screen pixels are clipped).

#### `drawHorizontalLine`

```zig
pub fn drawHorizontalLine(self: *Self, x: i16, y: i16, line_width: u8, color: Color) void
```

Draws a horizontal line starting at `(x, y)` extending `line_width` pixels to the right.

#### `drawVerticalLine`

```zig
pub fn drawVerticalLine(self: *Self, x: i16, y: i16, line_height: u8, color: Color) void
```

Draws a vertical line starting at `(x, y)` extending `line_height` pixels downward.

#### `drawRect`

```zig
pub fn drawRect(self: *Self, x: i16, y: i16, rect_width: u8, rect_height: u8, color: Color) void
```

Draws a rectangle outline. Does nothing if width or height is zero.

#### `fillRect`

```zig
pub fn fillRect(self: *Self, x: i16, y: i16, rect_width: u8, rect_height: u8, color: Color) void
```

Draws a filled rectangle.

#### `drawChar`

```zig
pub fn drawChar(self: *Self, x: i16, y: i16, character: u8, color: Color, font: Font) void
```

Draws a single character at the given position using the specified font.

#### `drawText`

```zig
pub fn drawText(self: *Self, x: i16, y: i16, text: []const u8, color: Color, font: Font) void
```

Draws a text string. Handles `\n` for line breaks and `\r` (ignored). Each character advances the cursor by `glyph_width + spacing_x` pixels.

#### `measureText`

```zig
pub fn measureText(self: *Self, text: []const u8, font: Font) u16
```

Returns the width in pixels that `text` would occupy when drawn with the given font, without actually drawing anything.

#### `present`

```zig
pub fn present(self: *Self) bool
```

Flushes the entire framebuffer to the display over I2C. Returns `true` on success. Call this after drawing to make changes visible on screen.

## Example

```zig
const avr = @import("avr_zig");
const i2c = avr.i2c;
const ssd1306 = avr.drivers.display.ssd1306;

const Oled = ssd1306.Display(128, 64);

pub fn main() void {
    i2c.init();

    var display = Oled{};
    if (!display.init()) return;

    display.clear(.off);
    display.drawText(0, 0, "HELLO", .on, ssd1306.default_font);
    display.drawRect(0, 0, 128, 64, .on);
    _ = display.present();
}
```
