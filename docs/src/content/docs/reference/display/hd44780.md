The HD44780 driver controls character LCD displays (such as the common 16x2 and 20x4 modules) connected through an I2C backpack based on the PCF8574 I/O expander.

```zig
const avr = @import("avr_zig");
const hd44780 = avr.drivers.display.hd44780_i2c;
```

:::note
You must call `avr.i2c.init()` before using this driver.
:::

## Constants

### `default_address`

```zig
pub const default_address: u7 = 0x27;
```

The most common I2C address for PCF8574-based backpacks.

### `alternate_address`

```zig
pub const alternate_address: u7 = 0x3F;
```

An alternate address used by some PCF8574A-based backpacks. If the display doesn't respond at `0x27`, try `0x3F` or run an [I2C scan](/reference/hal/i2c/).

## `Display`

```zig
pub fn Display(comptime display_columns: u8, comptime display_rows: u8) type
```

Returns a display driver type parameterized by screen dimensions. Common sizes are 16x2 and 20x4.

| Parameter | Type | Description |
|---|---|---|
| `display_columns` | `u8` | Number of character columns (comptime) |
| `display_rows` | `u8` | Number of rows (comptime) |

**Compile constraints:**

- Both dimensions must be greater than zero
- `display_rows` must be 4 or fewer

### Type constants

| Constant | Type | Description |
|---|---|---|
| `width` | `u8` | Number of columns |
| `height` | `u8` | Number of rows |
| `text_buffer_len` | `usize` | Size of the text buffer (`width * height`) |

### Fields

| Field | Type | Default | Description |
|---|---|---|---|
| `address` | `u7` | `0x27` | I2C address of the backpack |
| `backlight_enabled` | `bool` | `true` | Whether the backlight is on |
| `buffer` | `[text_buffer_len]u8` | all spaces | In-memory text buffer |

### Methods

#### `init`

```zig
pub fn init(self: *Self) bool
```

Runs the HD44780 initialization sequence (4-bit mode, display on, clear) and flushes the buffer. Returns `true` on success.

#### `clear`

```zig
pub fn clear(self: *Self) bool
```

Fills the text buffer with spaces, sends the clear-display command, and flushes the buffer. Returns `true` on success.

#### `home`

```zig
pub fn home(self: *Self) bool
```

Moves the hardware cursor to row 0, column 0 without changing the buffer contents.

#### `setBacklight`

```zig
pub fn setBacklight(self: *Self, enabled: bool) bool
```

Turns the backlight on or off.

| Parameter | Type | Description |
|---|---|---|
| `enabled` | `bool` | `true` for on, `false` for off |

#### `put`

```zig
pub fn put(self: *Self, column: u8, row: u8, char: u8) void
```

Stores a single character in the text buffer at the given position. Out-of-bounds coordinates are silently ignored. Characters outside printable ASCII (0x20--0x7E) are replaced with `'?'`. Call `present()` to send changes to the display.

#### `writeLine`

```zig
pub fn writeLine(self: *Self, row: u8, text: []const u8) void
```

Writes a string into one row of the text buffer, left-aligned. The rest of the row is filled with spaces. A `\n` in the text ends the line early.

| Parameter | Type | Description |
|---|---|---|
| `row` | `u8` | Row index (0-based) |
| `text` | `[]const u8` | Text to write |

#### `write`

```zig
pub fn write(self: *Self, start_column: u8, start_row: u8, text: []const u8) void
```

Writes text into the buffer starting at a specific position. Text wraps to the next row when it reaches the end of a line. Supports `\n` for explicit line breaks.

| Parameter | Type | Description |
|---|---|---|
| `start_column` | `u8` | Starting column (0-based) |
| `start_row` | `u8` | Starting row (0-based) |
| `text` | `[]const u8` | Text to write |

#### `present`

```zig
pub fn present(self: *Self) bool
```

Flushes the entire text buffer to the display hardware. Returns `true` on success. Call this after modifying the buffer with `put`, `writeLine`, or `write`.

#### `setCursor`

```zig
pub fn setCursor(self: *Self, column: u8, row: u8) bool
```

Moves the hardware cursor to a specific position. Returns `false` if the coordinates are out of bounds.

#### `writeChar`

```zig
pub fn writeChar(self: *Self, char: u8) bool
```

Writes a single character directly to the display at the current cursor position, bypassing the text buffer.

## Example

```zig
const avr = @import("avr_zig");
const i2c = avr.i2c;
const hd44780 = avr.drivers.display.hd44780_i2c;

const Lcd = hd44780.Display(16, 2);

pub fn main() void {
    i2c.init();

    var lcd = Lcd{};
    if (!lcd.init()) return;

    lcd.writeLine(0, "Hello, world!");
    lcd.writeLine(1, "AVR-Zig");
    _ = lcd.present();
}
```
